# Notes: Alternate-Transfer Escape Criterion

Date: 2026-03-09

## Objective

Freeze the standard for what counts as a genuine architecture escape in the scoped re-audit.

Without this, the re-audit can drift into cosmetic rewrites of the current negative route.

## Admissible Escape Standard

A candidate escape counts only if it does at least one of the following.

1. Produces a genuinely different integer transfer, still recognizably within the modern
   Kelley--Meka / Bloom--Sisask line, that removes or materially changes the recorded integer-case
   Bohr-set loss.
2. Replaces the structured-object step by a source-backed mechanism whose quantitative effect is
   strong enough to invalidate the current barrier abstraction rather than merely reparameterize
   it.
3. Supplies a source-backed theorem or explicit proof-line statement showing that one of the
   current critic abstractions is false or too coarse in a way that changes the exponent order.

## What Does Not Count

The following do not count as escapes:

- replacing global `L` by localized `L_j` inside the same recurrence body;
- rewriting the same terminal contradiction in supersaturation/counting language;
- proposing a completely different theorem technology and then claiming the current barrier is
  irrelevant;
- source-motivated optimism without a theorem-level quantitative consequence;
- transport rewrites that still pay the same effective Bohr/rank loss.

## Downstream Consequence Requirement

To be admissible, an escape must have a measurable downstream consequence in repository terms.

For example, it must threaten at least one of:

- the barrier formula `theta(a,b) = 1 / max(a+3, b+1)`;
- the current conclusion that the integer-case loss remains a real obstruction in the audited
  source set;
- the critic verdict that localization and endgame reformulation are exponent-order neutral inside
  the same architecture.

If none of those move, then the candidate is not a real escape.

## Criterion Verdict

```text
The scoped re-audit is now constrained to real architecture changes.
Cosmetic rewrites of the existing negative template are explicitly out of scope.
```
