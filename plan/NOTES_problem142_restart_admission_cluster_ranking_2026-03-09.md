# Notes: Erdős #142 Restart Admission Cluster Ranking

Date: 2026-03-09

## Objective

Rank the three admission clusters by immediate plausibility from the current post-Varnavides state.

## Ranking

1. `Cluster A`: benchmark-surviving transport audit
2. `Cluster B`: intermediate-debt mining
3. `Cluster C`: outside-family scan

## Reasons

### Why Cluster A is first

It is the most immediate admissibility test because:

- it acts directly on the one valid reduction object that was actually extracted;
- it asks one exact question with an exact failure mode;
- it is cheap to audit against the currently visible benchmark set.

Its weakness is already known:

```text
the existing visible benchmark inputs appear to lose exponent under proper subscale transport.
```

But that is exactly why it should be checked first: the audit can terminate quickly and honestly.

### Why Cluster B is second

It remains the best alternative if Cluster A fails, because:

- it still targets a smaller theorem debt below the direct `1 / 8` theorem;
- it stays close to the direct-counting fork rather than reopening the older exhausted family.

It is ranked second rather than first because the recent direct-counting verdict already found no
exact smaller supersaturation debt in the currently localized branch.

### Why Cluster C is third

It is last because:

- it is the broadest and least localized cluster;
- the earlier outside-family scouting already found that the non-counting candidates were too broad
  or too speculative to extract into one exact theorem-shaped question;
- reopening it first would risk recreating the same speculative sprawl the admission program is
  supposed to prevent.

## Verdict

```text
Run Cluster A first.

If Cluster A produces only a no-go verdict on the currently visible benchmark set, the admission
program should strongly prefer closure over opening a second speculative branch by default.
```
