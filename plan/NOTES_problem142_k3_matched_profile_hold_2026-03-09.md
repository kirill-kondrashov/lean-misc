# Notes: `k = 3` Matched-Profile Hold

Date: 2026-03-09

## Objective

Decide whether the stronger `k = 3` matched-profile route should remain an active live debt after
the existing source audits, or be placed on explicit hold.

## Existing Audited Source Status

### A. The sourced endpoint is still the `β = 1 / 12` split route

The Kelley--Meka audit concluded that the currently extracted source-backed upper theorem is only
strong enough for

```text
`β = 1 / 12`
```

not for

```text
`β > 1 / 2`.
```

So the honest source-backed `k = 3` endpoint remains the March 7 split pivot, not the old
matched-profile comparison route.

### B. The explicit `1 / 8` route is not source-backed

The March 8 `1 / 8` audit found no audited published theorem proving the explicit `1 / 8` bound.

So `1 / 8` remains conjectural/import-only scaffolding, not a source-backed theorem surface that
can reopen the route.

### C. The bottleneck note does not reopen the route

The bottleneck audit identifies a real local quantitative obstruction and explains why a localized
improvement target is mathematically meaningful.

But that note does not supply a new theorem.
And even a local `1 / 8` improvement would still sit below the old comparison threshold

```text
`β > 1 / 2`
```

used to close the matched-profile dominance target.

So the bottleneck note is evidence for future localized work, not a reason to keep the old
matched-profile closure as an active positive program.

## Reopen Condition

The stronger `k = 3` route may be reopened only if a genuinely audited source-backed theorem
appears that is strong enough to recover one of:

- `import_targets.k3_upper_exponent_gt_half_target`, or
- some mathematically equivalent direct comparison strong enough to yield
  `import_targets.split_gap_k3_profile_dominance_target`.

The following do not count as reopen evidence:

- source-motivated remarks about omitted optimizations,
- conjectural `1 / 8` scaffolding,
- or localized bottleneck diagnoses without a proved theorem statement.

## Hold Verdict

```text
Outcome: explicit hold.

The stronger `k = 3` matched-profile route is no longer an active positive theorem program.
The active honest `k = 3` endpoint remains the audited `β = 1 / 12` split route.
```

## Program Consequence

After this hold decision, the next remaining live debt is the scoped critic question:

```text
is there any genuinely source-backed alternate integer-transfer or structured-object route outside
the current negative template?
```

That should be handled by a periodic scoped re-audit, not by reviving the old positive `k = 3`
strengthening route without new sources.
