# Notes: Erdős #142 Post-Varnavides Program Revision

Date: 2026-03-09

## Objective

Revise the top-level research program after the quantitative Varnavides branch closed without
opening a smaller live theorem debt.

## Why A Revision Is Needed

The March 8 revised program was a successful reset away from packaging drift, but its original
active tracks are now historical:

- the frontier debt ledger is complete;
- the source-first audit chain is complete;
- the route-generation program is complete;
- the post-dormancy exact-debt extraction succeeded once, then localized and closed in the
  Varnavides branch.

So the top-level plan should no longer read like an active debt-reduction engine.

## New Program Mode

The correct mode is:

```text
dormant, with explicit admission gates for any future restart.
```

This means:

- no theorem-construction cycle is active by default;
- a new cycle opens only after one exact smaller debt is identified;
- the burden of proof is on the new candidate route, not on the plan to keep moving.

## Admissible Reopen Channels

### Channel A. Benchmark-surviving transport input

Find a concrete upper input that, when pushed through the Varnavides transport inequality, yields
a genuinely smaller debt at scale `N` without circular same-scale reuse.

### Channel B. Smaller direct-counting intermediate debt

Find an exact supersaturation, stability, or counting statement that is strictly smaller than the
full direct `1 / 8` theorem and not just a reformulation of the current transport inequality.

### Channel C. Exact debt outside the exhausted family

Find a new source-backed or theorem-shaped target outside the current split / matched-profile /
Bohr-recurrence / direct-counting restart branches.

## Disallowed Pseudo-Progress

Do **not** reopen the program with:

- another generic reduction wrapper;
- another same-scale restatement of an existing benchmark;
- another checker / README / theorem-surface cycle;
- another conjectural import route without new audited source support.

## Verdict

```text
The top-level research program should now be read as a dormant governance document.

Its job is to preserve stop rules and define the only admissible restart channels, not to imply
that a live theorem-construction cycle currently exists.
```
